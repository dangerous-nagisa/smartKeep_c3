## 2026-04-02 | BMI270 姿态解算管道设计与实现

### 1. 任务背景

在 ESP32-C3 平台上实现基于 BMI270 六轴 IMU 的实时姿态解算系统，输出标准化数据包 `node_packet_t` 供后续无线传输使用。

**硬件约束：**
- ESP32-C3：RISC-V 单核 160MHz，无硬件 FPU
- BMI270：Bosch 六轴 IMU（加速度计 + 陀螺仪），无磁力计

**引脚分配：**
| GPIO | 功能 | 备注 |
|:---|:---|:---|
| GPIO4 | I2C SDA | BMI270 数据线 |
| GPIO5 | I2C SCL | BMI270 时钟线 |
| GPIO2 | ADC | 电池电压采样 (TODO) |
| GPIO6 | PWM | 蜂鸣器 (TODO) |

---

### 2. 数据包定义

```c
typedef struct {
    uint8_t  node_id;       // 节点 ID
    uint8_t  battery;       // 电池电量 (0~100, 0xFF=未知)
    uint16_t seq;           // 包序列号
    uint32_t timestamp;     // 启动以来毫秒数 (溢出周期 ~49天)

    float    euler[3];      // Roll / Pitch / Yaw (度)

    float    accel[3];      // ax / ay / az (m/s², 滤波后)
    float    gyro_norm;     // |ω| 角速度模值 (°/s)

    float    jerk;          // |da/dt| 加加速度 (m/s³)
    uint8_t  step_flag;     // 1 = 本帧检测到一步
    uint8_t  crc8;          // CRC-8 校验 (poly=0x07)
} node_packet_t;            // sizeof = 44 字节
```

**带宽估算：** 44 字节 × 50 Hz = 2.2 KB/s，BLE/ESP-NOW 均可承载。

### 4. 算法设计

#### 4.1 陀螺仪零偏校准

**问题：** MEMS 陀螺仪存在零点漂移，静止时输出非零值。

**方案：**
- 上电后静置 ~2 秒，采集 200 帧陀螺仪数据
- 计算三轴平均值作为零偏 `bias_gx/gy/gz`
- 后续每帧减去零偏

```c
#define GYRO_CALIB_SAMPLES  200

// 校准阶段
while (n < GYRO_CALIB_SAMPLES) {
    sum_gx += lsb_to_dps(sd.gyr.x);
    // ...
}
bias_gx = sum_gx / n;
```

#### 4.2 一阶 IIR 低通滤波

**问题：** IMU 高频噪声影响姿态解算稳定性。

**方案：** 指数移动平均 (EMA)

```c
filtered = α × new + (1 - α) × old
```

**参数选择：**
| 信号 | α | 截止频率 (近似) | 理由 |
|:---|:---|:---|:---|
| 加速度 | 0.3 | ~5 Hz | 姿态变化慢，多平滑 |
| 陀螺仪 | 0.5 | ~16 Hz | 需保留快速旋转信息 |

#### 4.3 Madgwick AHRS (6轴)

**选型理由：**
| 算法 | 优点 | 缺点 | 适用性 |
|:---|:---|:---|:---|
| 互补滤波 | 简单 | 需手调权重，精度一般 | 快速原型 |
| Madgwick | 计算量小、收敛快、精度好 | 无磁力计 Yaw 漂移 | **本项目选用** |
| Mahony | 类似 Madgwick | 需调两个参数 | 可选 |
| EKF/UKF | 理论最优 | 计算量大 | 资源充足时 |

**核心公式：**
```
四元数微分 = 0.5 × q ⊗ ω           (陀螺仪积分)
梯度校正   = β × ∇f(q, a)          (加速度计校正)
q_new      = q + (qDot - 梯度校正) × dt
```

**β 参数策略：**
| 阶段 | β | 说明 |
|:---|:---|:---|
| 快速收敛 (前 1s) | 2.0 | 加速度计高权重，快速对齐重力 |
| 稳态运行 | 0.04 | 陀螺仪主导，减少抖动 |

**欧拉角计算 (ZYX 顺序)：**
```c
roll  = atan2(2(q0q1 + q2q3), 1 - 2(q1² + q2²))
pitch = asin(2(q0q2 - q3q1))
yaw   = atan2(2(q0q3 + q1q2), 1 - 2(q2² + q3²))
```

#### 4.4 Jerk 计算

```c
jerk = |da/dt| = sqrt((Δax/Δt)² + (Δay/Δt)² + (Δaz/Δt)²)
```

用于检测突然的加速度变化（跌倒、碰撞等场景）。

#### 4.5 简易计步检测

**算法：** 加速度模值阈值 + 去抖

```c
#define STEP_ACCEL_THRESH   12.0f   // m/s² (略大于 1g)
#define STEP_MIN_INTERVAL   300     // ms

if (|a| > 阈值 && 距上次 > 间隔) {
    step_flag = 1;
}
```

**局限性：** 简单阈值法，适合初步验证，后续可升级为峰值检测或机器学习方案。

## 2026-04-02 | UDP 数据传输模块设计与实现

### 1. 任务背景

为姿态解算节点添加无线数据传输能力，将 `node_packet_t` 通过 WiFi UDP 广播发送，供上位机或其他设备接收。

**需求：**
- WiFi STA 模式连接热点
- UDP 广播发送 `node_packet_t`（44 字节 @ 50 Hz）
- 热点 SSID/密码留空，用户自行配置

---

### 2. 系统架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                            系统数据流                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌──────────┐      ┌─────────────┐      ┌──────────────┐              │
│   │ imu_task │ ──►  │ FreeRTOS    │ ──►  │ udp_tx_task  │              │
│   │ (50 Hz)  │      │ Queue (10)  │      │ sendto()     │              │
│   └──────────┘      └─────────────┘      └──────────────┘              │
│        │                                        │                       │
│        │ node_packet_t                          │ UDP                   │
│        │ 44 bytes                               ▼                       │
│        │                              ┌──────────────────┐              │
│        │                              │ 255.255.255.255  │              │
│        ▼                              │     :8888        │              │
│   串口打印 (1 Hz)                      └──────────────────┘              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**任务解耦：**
- `imu_task`：IMU 读取 + 姿态解算 + 打包 → 队列（不阻塞）
- `udp_tx_task`：队列取数据 → UDP 发送（阻塞等待）
- 队列满时 IMU 侧丢弃最新包，保证实时性

---

### 3. WiFi STA 配置

```c
/* --- WiFi 配置 --- */
#define WIFI_SSID               ""          // ← 用户填入热点名称
#define WIFI_PASS               ""          // ← 用户填入热点密码
#define WIFI_MAX_RETRY          10
```

**初始化流程：**
```
nvs_flash_init()
    │
    ▼
esp_netif_init() + esp_event_loop_create_default()
    │
    ▼
esp_netif_create_default_wifi_sta()
    │
    ▼
esp_wifi_init() + 注册事件回调
    │
    ▼
esp_wifi_set_config(SSID, PASS)
    │
    ▼
esp_wifi_start() → 自动触发连接
    │
    ▼
等待 EventGroup: WIFI_CONNECTED_BIT | WIFI_FAIL_BIT
```

**断连处理：**
- 自动重连，最多 `WIFI_MAX_RETRY` 次
- 超过重试次数后设置 `WIFI_FAIL_BIT`
- 主程序检测到连接失败后，仍启动 IMU，仅串口输出

---

### 4. UDP 传输设计

```c
/* --- UDP 配置 --- */
#define UDP_BROADCAST_IP        "255.255.255.255"
#define UDP_PORT                8888
```

**udp_tx_task 流程：**
```
创建 socket (SOCK_DGRAM)
    │
    ▼

    │
    ▼

    │

    │
    └─ sendto(广播地址, 44字节)
```

**关键参数：**
| 参数 | 值 | 说明 |
|:---|:---|:---|
| 协议 | UDP | 尽力而为，低延迟 |
| 目标地址 | 255.255.255.255:8888 | 局域网广播 |
| 包大小 | 44 字节 | `sizeof(node_packet_t)` |
| 频率 | 50 Hz | 2.2 KB/s |
| 队列深度 | 10 | ~200ms 缓冲 |

### 5. FreeRTOS 任务配置

| 任务 | 栈大小 | 优先级 | 职责 |
|:---|:---|:---|:---|
| `imu_task` | 4096 | 5 | IMU 读取 + 姿态解算 + 打包 |
| `udp_tx_task` | 3072 | 4 | 队列消费 + UDP 发送 |

**优先级策略：**
- IMU 优先级 > UDP：保证姿态解算实时性
- UDP 任务阻塞在队列上，不抢占 IMU

**步骤 3：接收数据**

PC 端 Python 示例：
```python
import socket
import struct

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sock.bind(('0.0.0.0', 8888))

# node_packet_t 解析格式 (44 字节)
PKT_FMT = '<BBHIffffff f fBB'  # 小端序

while True:
    data, addr = sock.recvfrom(64)
    if len(data) == 44:
        fields = struct.unpack(PKT_FMT, data)
        node_id, battery, seq, timestamp = fields[:4]
        roll, pitch, yaw = fields[4:7]
        ax, ay, az = fields[7:10]
        gyro_norm = fields[10]
        jerk = fields[11]
        step_flag, crc8 = fields[12:14]
        
        print(f"[{seq:5d}] R={roll:+6.1f} P={pitch:+6.1f} Y={yaw:+6.1f}")
```

## 2026-04-02 | 步态检测状态机设计与实现

### 1. 需求

将 `step_flag` 从简单的"检测到一步"升级为**触地/离地双事件检测**，服务于医学步态评估场景。

**核心思路：**
- `jerk`（加加速度）：衡量加速度变动速度，用于检测冲击（触地）
- `gyro_norm`（角速度模值）：检测甩腿动作（离地）
- `accel`（加速度）：辅助判断支撑/摆动状态

---

### 2. step_flag 编码

| 值 | 含义 | 英文 | 物理现象 |
|:---|:---|:---|:---|
| **0** | 无事件 | NONE | 正常运动 |
| **1** | 触地 | Heel Strike (HS) | 脚跟砸向地面，jerk 尖峰 + 加速度冲击 |
| **2** | 离地 | Toe Off (TO) | 脚尖蹬离地面，角速度飙升 + 加速度回落 |

---

### 3. 状态机设计

```
             ┌──────────────────────────────────────────────────┐
             │                                                  │
             ▼                                                  │
        ┌─────────┐    触地条件满足     ┌──────────┐            │
  ──►   │  SWING  │ ──────────────────► │ STANCE   │            │
        │ (摆动期) │   step_flag = 1    │ (支撑期)  │            │
        │ 脚在空中 │   Heel Strike      │ 脚在地上  │            │
        └─────────┘                     └──────────┘            │
             ▲                               │                  │
             │         离地条件满足           │                  │
             └──────────────────────────────┘                  │
                       step_flag = 2                           │
```

### 4. 检测条件

#### 4.1 触地检测 (SWING → STANCE)

```
条件 = (jerk > 50 m/s³ 或 |accel| > 14.7 m/s²)
      且 距上次事件 > 150ms
      且 距上次触地 > 300ms
```

| 信号 | 阈值 | 换算 | 理由 |
|:---|:---|:---|:---|
| `jerk` | > 50 m/s³ | ~5 g/s | 脚跟撞击地面的加加速度尖峰 |
| `|accel|` | > 14.7 m/s² | ~1.5 g | 地面反作用力冲击 |

#### 4.2 离地检测 (STANCE → SWING)

```
条件 = gyro_norm > 100 °/s
      且 |accel| < 11.0 m/s²
      且 距上次事件 > 150ms
```

| 信号 | 阈值 | 理由 |
|:---|:---|:---|
| `gyro_norm` | > 100 °/s | 腿部甩动时的角速度峰值 |
| `|accel|` | < 11.0 m/s² | 离地后失去地面支撑力，加速度回落到接近 1g |

| `STEP_MIN_INTERVAL_MS` | 300 ms | 两次触地之间最小间隔 (~3.3 步/秒上限) |

## 2026-04-02 | 5节点复用系统设计与实现

### 1. 需求背景

系统需支持 **5 个 IMU 传感器节点** 同时工作，用于临床步态评估（TUG、10MWT、5xSTS 等）。

**核心需求：**
- 同一固件烧录所有板子，节点 ID 运行时可配置
- 节点是通用硬件单元，身体位置映射由接收端模板决定
- 远程修改 ID，无需物理接触板子（适合佩戴后调整）

---

### 2. 设计决策

| 问题 | 方案 | 理由 |
|:---|:---|:---|
| ID 存储 | **NVS Flash** | 掉电保持，一次配置永久有效 |
| ID 配置方式 | **UDP 远程配置** | 板子装在身上时可通过 PC 修改 |
| 碰撞处理 | **不处理** | 250 pkt/s 远低于 WiFi 带宽，CSMA/CA 已内置 |
| 位置映射 | **接收端模板** | 同一套硬件可适配多种评估场景 |

### 4. UDP 配置协议

**端口：** 8889

**请求包 (6 字节)：**
```
┌──────────┬─────────┬─────────┐
│ "SKCF"   │  cmd    │  value  │
│ (4B)     │  (1B)   │  (1B)   │
└──────────┴─────────┴─────────┘
```

**应答包 (6 字节)：**
```
┌──────────┬─────────┬─────────┐
│ "SKCK"   │  cmd    │  value  │
│ (4B)     │  (1B)   │  (1B)   │
└──────────┴─────────┴─────────┘
```

| cmd | 名称 | 说明 |
|:---:|:---|:---|
| 0x01 | SET_ID | value=1~5，设置节点 ID 并保存 NVS |
| 0x02 | GET_ID | 查询当前 ID，应答 value 返回 ID |

**错误码：** value=0xFF 表示无效 ID

### 9. 关键实现细节

#### NVS 存储

```c
// 命名空间: "smartkeep"
// 键: "node_id"
// 值: uint8_t (1~5)

nvs_handle_t h;
nvs_open("smartkeep", NVS_READWRITE, &h);
nvs_set_u8(h, "node_id", id);
nvs_commit(h);
nvs_close(h);
```

#### UDP 配置任务

- 独立 FreeRTOS 任务，栈 2560 字节，优先级 3
- 阻塞等待配置请求，收到后立即回复 ACK
- SET_ID 成功后同时更新内存变量和 NVS

#### main.c 初始化顺序

```
1. NVS 初始化
2. node_config_init() — 加载 ID
3. WiFi 连接
4. node_config_start_udp_listener() — 启动配置监听
5. 创建数据队列
6. 启动 UDP 发送任务
7. I2C / BMI270 初始化
8. 启动 IMU 任务
```
